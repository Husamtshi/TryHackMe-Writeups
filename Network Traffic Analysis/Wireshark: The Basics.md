# Key Concepts

## What is Wireshark?

Wireshark is one of the most widely used network protocol analyzers.

It allows analysts to:

- Capture live network traffic.
- Analyze PCAP files.
- Inspect individual packets.
- Troubleshoot network issues.
- Investigate cyber security incidents.

Wireshark operates at the packet level, making it one of the most powerful tools for network analysis.

---

## Wireshark Interface

The Wireshark interface consists of three main sections:

### Packet List

Displays all captured packets.

Useful information includes:

- Packet Number
- Timestamp
- Source IP
- Destination IP
- Protocol
- Packet Length
- Additional Information

---

### Packet Details

Displays protocol layers contained inside the selected packet.

Examples include:

- Ethernet
- IP
- TCP / UDP
- HTTP
- DNS
- TLS

Each protocol can be expanded to inspect its individual fields.

---

### Packet Bytes

Displays the raw packet in:

- Hexadecimal
- ASCII

This allows analysts to inspect packet contents at the byte level.

---

## Packet Dissection

Wireshark automatically breaks packets into protocol layers.

This process is called **Packet Dissection**.

Instead of viewing raw binary data, analysts can easily inspect each protocol separately.

---

## Packet Navigation

Efficient navigation is essential when investigating large PCAP files.

Useful navigation techniques include:

- Jumping between packets.
- Following conversations.
- Expanding protocol layers.
- Inspecting packet timestamps.
- Comparing related packets.

---

## Packet Filtering

Display filters help analysts isolate relevant traffic without modifying the original capture.

Common examples include:

```text
ip.addr == 192.168.1.10
```

```text
tcp
```

```text
udp
```

```text
dns
```

```text
http
```

```text
icmp
```

Filters allow analysts to focus only on the traffic relevant to their investigation.

---

# Key Takeaways

- Wireshark is the industry-standard tool for packet analysis.
- Every packet contains multiple protocol layers.
- Understanding packet structure is essential before performing threat hunting.
- Display filters greatly simplify traffic analysis.
- Efficient navigation saves significant investigation time.

---

# SOC Perspective

Wireshark is one of the primary tools used by SOC analysts during network investigations. It enables analysts to inspect network traffic at the packet level, identify suspicious communications, investigate malware activity, analyze protocols, and validate alerts generated by IDS or SIEM platforms. Mastering the basics of packet navigation and filtering provides the foundation for more advanced network threat hunting.
